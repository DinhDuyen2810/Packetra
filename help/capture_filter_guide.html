Capture Filter Guide

This page covers capture filters only. These filters are applied before live packets are brought into Packetra. They use BPF/libpcap-style syntax, not Packetra display-filter syntax.

1. Capture filter vs display filter

TypeWhen it appliesSyntax familyMain effect
Capture filterBefore packets are capturedBPF / libpcap styleReduces what gets recorded from the wire
Display filterAfter packets already exist in PacketraPacketra display-filter engineReduces what is shown for analysis

2. Every in-app place where you can set a capture filter

  1. Capture Options / Input tab: the field labeled Capture filter for selected interfaces:.
  2. Capture Options / Input tab: the Capture Filters... button opens the preset manager.
  3. Interface selector view: the quick capture_filter_input field where you can type a capture filter before pressing start.
  4. Main menu: Capture > Capture Filters... opens the preset manager from the main application window.
  5. Remote capture flow: the chosen interface carries its configured capture filter into the remote start command.

3. Full UI workflow inside Capture Options

  1. Open Capture > Options....
  2. In the Input tab, select one top-level interface row in the interface tree.
  3. Use the field Capture filter for selected interfaces: to type the filter.
  4. Click Apply to bind that filter to the currently selected interface row.
  5. The assigned filter is stored in the interface tree’s Capture Filter column.
  6. If you switch interface selection, Packetra synchronizes the filter input field with the selected interface’s saved capture filter.
  7. Double-clicking an interface or pressing start uses that interface’s assigned capture filter.

Important: capture filters are per selected interface row in this workflow, not a universal live global filter for every interface at once.

4. Capture filter preset manager

The preset manager is shared between the capture options flow and the main menu capture-filter dialog.

  • Open it using Capture Filters....
  • Create named presets with expression and optional comment.
  • Packetra validates syntax through libpcap-compatible Scapy compilation backends when available.
  • After closing the dialog with a selected preset, the selected preset expression can be copied back into the current interface filter field.
  • Preset names can act as aliases, because Packetra resolves a preset name to its stored expression before starting capture.

5. Capture filter validation behavior

  • Packetra tries to validate with scapy.arch.pcapdnet.compile_filter.
  • If that backend is unavailable, it tries scapy.all.compile_filter.
  • If compilation succeeds, the filter is accepted.
  • If compilation fails, Packetra shows the returned syntax error.

This means Packetra aims to support advanced BPF syntax rather than only a small simplified subset.

8. How capture filters are used in local vs remote capture

  • Local capture: Packetra passes the validated capture filter to the packet sniffer object.
  • Remote Linux capture: Packetra appends the BPF filter to the remote tcpdump command.
  • Remote Windows capture: Packetra passes the filter to the remote capture agent command.

Because the same filter text may be executed on different backends, simple and standard BPF syntax is the safest choice when moving between environments.

9. Best practice

  • Use capture filters only when you are sure about the traffic boundary.
  • If you are still exploring, capture broader traffic and use display filters later.
  • For remote hosts, keep the capture filter conservative first so you can confirm the right interface and permissions.
  • If capture returns nothing, remove the capture filter entirely and test again.
