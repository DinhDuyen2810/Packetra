Packetra Help Center

This help center is meant to be practical, not superficial. Each page focuses on what you can actually do in the current Packetra build: capture traffic, apply display filters, use dashboards, and work with remote capture from Linux or Windows hosts.

3. Recommended Reading Order

  1. If you are new to Packetra, begin with User Guide.
  2. If your main pain point is filtering, jump directly to Display Filter Reference.
  3. If you need BPF-style input before packets are captured, open Capture Filter Guide.
  4. If you need to work with a Linux server instead of a local interface, open Remote Capture Guide.
  5. If you want summaries and visual investigation instead of raw packet-by-packet reading, open Dashboard Guide.

4. What This Help Set Covers

  • The menu structure Packetra exposes today: capture, analysis, statistics, tools, and help.
  • The display filter syntax implemented in core/filtering.py, including supported protocol aliases and field names.
  • The capture filter preset manager and per-interface capture-filter assignment flow implemented in the capture options UI.
  • The remote capture flow implemented through SSH in core/remote_capture.py and the remote interface manager dialog.
  • The dashboard system backed by packet, endpoint, conversation, protocol, DNS, and HTTP-oriented data sources.

5. Useful Companion Menu Items

  • Help > System and Version Information shows Packetra version, Python version, Npcap status, and installed library versions.
  • Analyze > Display Filter Expression opens the filter expression builder with the in-app field catalog.
  • Tools > Dashboard opens the dashboard overview and template gallery.
  • Tools > AI Analyst remains available for local traffic interpretation, while the remote page focuses on capture from real servers.
