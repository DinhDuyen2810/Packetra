Packetra User Guide

This is the complete user-facing guide for the current Packetra build. It is designed as an operator manual: what the application contains, what each major screen and command does, and how to move from raw traffic to a finished analysis session.

2. Startup and First Decisions

2.1 Live capture vs file analysis

ModeUse it whenTypical entry point
Live captureYou need to observe traffic happening now on a local or remote interface.Select an interface and press Start.
File analysisYou already have a `.pcap` or `.pcapng` capture for offline investigation.File > Open...

2.2 Windows and Npcap

On Windows, local live capture depends on Npcap. If Packetra cannot use Npcap, live capture on local Windows interfaces will not start. You can verify the environment under Help > System and Version Information.

Remote capture to a Linux server is different: it relies on SSH and tcpdump on the remote host, not local Npcap.

3. Main Interface Layout

3.1 Menu bar

The menu bar is the full command surface of Packetra. It contains the primary application actions: file handling, capture, navigation, analysis, statistics, advanced tools, and help.

3.2 Main toolbar

The toolbar gives quick access to common operations such as start, stop, restart, capture options, open, save, reload, search, packet navigation, stay-at-end mode, zoom, resize columns, and layout reset.

3.3 Packet panes

  • Packet list: the high-level packet table showing packet number, time, source, destination, protocol, length, and info.
  • Packet details: the tree representation of protocols and fields for the currently selected packet.
  • Packet bytes: the byte and hex view used for payload verification and low-level inspection.
  • Packet diagram placeholder: a reserved packet-diagram area that is present in the layout control model.

3.4 Status bar

The status bar reflects the current mode of the session: load activity, capture activity, selected frame information, filtered state, packet counts, dropped counts, and quick access buttons such as expert information or capture file properties.

4. Capturing Traffic

4.1 Local capture

  1. Choose a local interface from the interface selection screen or capture options dialog.
  2. If needed, assign a capture filter to that interface.
  3. Choose whether promiscuous mode should be enabled.
  4. Start capture.
  5. Watch packets arrive live in the packet list.
  6. Stop or restart capture when needed.

4.2 Remote capture

Packetra also supports remote interfaces. The remote host and interface exposure are managed through Manage Interfaces. Linux-focused remote guidance is documented in Remote Capture Guide.

4.3 Open an existing file

Use File > Open... to load a file. Packetra supports saved capture workflows including reload, merge, save, save as, separate packets, export selected scopes, and print-related flows.

5. Full Menu Reference

5.1 File menu

  • Open...: load a capture file.
  • Merge...: merge another file into the current analysis flow.
  • Save and Save As...: persist the current capture.
  • Separate: split out packets into new capture outputs by selected logic.
  • Export Specified Packets...: export a chosen subset of packets.
  • Print...: print-oriented output flow.
  • Quit: exit the application, with save prompts when appropriate.

5.2 Edit menu

  • Copy: copy selected content depending on current context.
  • Find Packet..., Find Next, Find Previous: find workflow across packet list, details, or bytes.
  • Mark/Unmark Selected, Mark/Unmark All Displayed Packets, Next Mark, Previous Mark: annotation and navigation over marked packets.
  • Ignore/Unignore Selected, Ignore/Unignore All Displayed: remove or restore packets from active analytical visibility.
  • Packet Comment..., Delete All Packet Comments: comment system for per-packet notes.
  • Preferences...: application configuration and customization.

5.3 View menu

  • Show or hide the main toolbar, filter toolbar, and status bar.
  • Show or hide the packet list, packet details, packet bytes, and packet diagram panes.
  • Zoom in, zoom out, and reset to normal text size.
  • Expand or collapse selected subtrees and all subtrees in the details pane.
  • Enable packet coloring and conversation coloring, and open coloring-rule configuration.
  • Resize all columns, show a packet in a new window, redissect packets, reload as file-format view, or reload the current file.

5.4 Go menu

  • Back and Forward for navigation state.
  • Go to Packet and Go to Corresponding Packet.
  • Previous / Next / First / Last packet navigation.
  • Previous / Next packet in conversation.
  • Auto Scroll in Live Capture to keep the newest traffic visible during capture.

5.5 Capture menu

  • Options...: open capture configuration.
  • Start, Stop, Restart: primary live-capture controls.
  • Capture Filters...: open the capture-filter preset manager.
  • Refresh Interfaces: refresh the available interface list when not actively capturing.

5.6 Analyze menu

  • Display Filter Macros...: reusable named display-filter snippets.
  • Display Filter Expression...: builder for field/operator/value display filters.
  • Apply as Column: turn a selected field into a visible analytical column.
  • Apply as Filter: generate a display filter from the selected field.
  • Conversation Filter: build a conversation-scoped filter from the current packet.
  • Follow: follow TCP or UDP stream / conversation data flow.
  • Expert Info: open the expert-information window for warnings and notable observations.

5.7 Statistics menu

  • Capture File Properties
  • Resolved Addresses
  • Protocol Hierarchy
  • Conversations
  • Endpoints
  • Packet Lengths
  • Flow Graph
  • HTTP
  • IPv4 Statistics
  • IPv6 Statistics

5.8 Tools menu

  • AI Analyst: AI-assisted interpretation of selected traffic.
  • Demo Packet: sample/demo exploration flow.
  • Network Topology Graph: graph-based communication visualization.
  • Dashboard: dashboard gallery and template-driven overview system.
  • Firewall ACL Rules: generate firewall rule suggestions from a selected packet.

5.9 Help menu

  • System and Version Information: Packetra, Python, Npcap, and dependency versions.
  • HTML documentation pages for user guide, capture workflow, capture filters, display filters, dashboard usage, and remote capture.
  • About Packetra and About Qt.

6. Packet Analysis Workflow

6.1 Filter

Packetra uses a display-filter workflow after packets have already been loaded or captured. You can type directly into the filter bar, use right-click filter helpers, drag certain packet-list cells into the filter bar, use macros, use the expression builder, or push filters from statistics and topology views. For reusable snippets, open Analyze > Display Filter Macros... and then reference the macro name in later filters.

For the full filter mechanics, use Display Filter Reference. For pre-capture filtering, use Capture Filter Guide.

6.2 Inspect details and bytes

  • Select a packet in the packet list.
  • Inspect protocol layers and fields in the details tree.
  • Use expand, collapse, or copy actions from the detail context menu.
  • Inspect packet bytes in the byte pane and use byte-selection linking with details when needed.

6.3 Follow streams and conversations

For supported packets, Packetra can open a Follow dialog with view modes such as full conversation or directional stream output. From there you can:

  • Change rendering format such as ASCII, Hex Dump, or Base64-like raw output.
  • Restrict to client-to-server or server-to-client direction.
  • Filter this stream, filter out this stream, copy the rendered text, or save it.

7. Statistics Windows

7.1 Addresses and hierarchy

  • Resolved Addresses: map MAC OUIs and vendor names, with search and optional limit-to-display-filter behavior.
  • Protocol Hierarchy: show protocol breakdown with percentages, bytes, and optional Apply as Filter for selected protocols.

7.2 Endpoints and conversations

  • Conversations: analyze peer-to-peer exchanges.
  • Endpoints: tabbed breakdown by Ethernet, IPv4, IPv6, TCP, and UDP endpoints.

7.3 Packet lengths and flow graph

  • Packet Lengths: inspect packet-size distribution and related summary metrics.
  • Flow Graph: sequence-oriented flow inspection with row selection, apply-filter options, and navigation back to packets.

7.4 HTTP, IPv4, IPv6

  • HTTP Packet Counter: request/response counts and HTTP status-class grouping.
  • IPv4 Statistics: address-based packet and burst summaries with filter push-back.
  • IPv6 Statistics: same style of analytical breakdown for IPv6 addresses.

8. Advanced Tools

8.1 AI Analyst

AI Analyst is a guided interpretation tool for selected traffic. It should be used as an assistant, not as final proof. Its output should be verified with packet fields, bytes, conversation scope, and statistical context.

8.2 Demo Packet

Demo Packet provides a curated or sample-style exploration path to help test the analysis UI or demonstrate packet interpretation features.

8.3 Network Topology Graph

The topology graph visualizes relationships between communicating nodes. It also provides filter-related actions such as Apply Filter and Copy Filter based on the selected node or edge.

8.4 Dashboard

The dashboard system gives template-based summarized views over the currently displayed packet scope. It includes templates such as Network Overview, DNS Analysis, Timeline Analysis, HTTP/TLS Analysis, Protocol Analysis, Endpoint Activity, Security Investigation, and Topology View.

8.5 Firewall ACL Rules

When exactly one suitable packet is selected, Packetra can generate firewall rule suggestions for products such as Cisco, iptables, pf, ipfw, and Windows netsh variants. The generated content is a template that must still be reviewed before real deployment.

9. Preferences and Customization

9.1 Appearance and filter UX

  • Display-filter autocomplete behavior.
  • How many recent filter entries to show.
  • Fonts and colors for packet list, details, bytes, marked packets, ignored packets, and search highlighting.

9.2 Layout and pane behavior

  • Default pane layout and restore-layout behavior.
  • Visibility of major panes and toolbar components.
  • Column sizing and reset actions.

9.3 Capture behavior

  • Promiscuous mode defaults.
  • Capture-output file naming, format, rollover, compression, and ring-buffer settings.
  • Related real-time and save/output behavior.

10. End-to-End Example Session

  1. Start Packetra.
  2. Choose whether to work with a live interface or an existing capture file.
  3. If capturing live, assign an optional capture filter and start capture.
  4. Watch the packet list fill with traffic.
  5. Apply a broad display filter such as tcp, dns, or http or tls.
  6. Select representative packets and inspect details and bytes.
  7. Use Follow or Conversation Filter if you need stream-level focus.
  8. Open Protocol Hierarchy, Endpoints, or Dashboard for summary context.
  9. If needed, use AI Analyst or Topology Graph to get alternate views of the same traffic.
  10. Save the capture or export the subset that matters.

11. Common Mistakes and Advice

  • Do not confuse capture filters with display filters.
  • Do not rely on AI output without technical verification.
  • Do not over-filter too early in a new investigation.
  • Do not forget that many statistics and tool windows operate on the currently displayed packet scope.
  • Do not close or replace an important capture before deciding whether it should be saved.
Best operator pattern: capture or open data first, filter second, verify with details and bytes third, summarize with statistics/dashboard fourth, and save/export last.
